Four ways to write the same number

Every number system uses place value — digits mean more the further left they sit. What changes is the base: how many symbols exist before you have to carry over. Tap a card to see it broken down.

2
Binary
Base 2 · the computer's native tongue
▾

Only two symbols exist, so every place value is a power of 2. Circuits only need to sense "on" or "off" — which is exactly what a bit represents.

01
1101₂
1×2³+1×2²+0×2¹+1×2⁰ =8+4+0+1=13
💡Used for: logic gates, RAM, file storage, network packets — anything a transistor decides yes/no about.
8
Octal
Base 8 · a tidy shorthand for binary
▾

Eight symbols, so every place is a power of 8. Because 8 = 2³, each octal digit maps neatly onto exactly 3 binary bits — a compact way to write long bit strings by hand.

0123 4567
157₈
1×8²+5×8¹+7×8⁰ =64+40+7=111
💡Used for: classic Unix file permissions (chmod 755), older systems' memory dumps.
10
Decimal
Base 10 · the one humans grew up on
▾

Ten symbols — almost certainly because we have ten fingers. Every place value is a power of 10, which is why "carrying the 1" happens once you pass 9.

01234 56789
243₁₀
2×10²+4×10¹+3×10⁰ =200+40+3=243
💡Used for: everyday counting, money, measurements — every non-technical context.
16
Hexadecimal
Base 16 · binary's favourite disguise
▾

Ten digits aren't enough once you pass 9, so hex borrows letters A–F to mean 10–15. Since 16 = 2⁴, one hex digit always equals exactly 4 bits — a huge readability win over raw binary.

01234 56789 A=10B=11C=12D=13E=14F=15
2F₁₆
2×16¹+15×16⁰ =32+15=47
💡Used for: colour codes like #FF6600, memory addresses, MAC addresses, debugging output.
